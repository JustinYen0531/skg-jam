# SKG: Scorekeeper Implementation Checklist

> 這份清單是 GDD 與 prototype 之間的修正主線。每次修改前後都要檢查它，完成一項就更新狀態與備註。
>
> 狀態：`[ ]` 待處理、`[~]` 進行中、`[x]` 已完成、`[!]` 被阻塞或需要重新設計

## 工作流規則

- 一次只處理一個主要修正，避免同時改散多條主線。
- 每次開始修改前，先檢查本清單，確認目前要處理的項目與驗收條件。
- 每次修改完成後，必須重新檢視本清單，更新完成狀態與實際結果。
- 每次修改後至少執行 `npm run lint` 與 `npm run build`。
- 不得擅自開啟 preview 或瀏覽器驗證；預設使用自動測試、lint 與 build，只有使用者當次明確同意才可開啟預覽。
- 如果發現修改影響其他清單項目，要在「備註／驗證紀錄」留下註記。
- 沒有完成的項目不能標記為完成；若只完成一部分，使用 `[~]` 並寫清楚剩餘差距。
- 每次完成一個工作單位後，必須 `git add`、`git commit`、`git push` 到 GitHub，不能只保留本機修改。

- [x] 保留現有左側面板程式碼，不直接刪除，改為開發者專用工具。
- [x] 支援 `Ctrl + Shift + D` 切換 Debug 面板。
- [x] 支援網址參數 `?debug=true` 直接開啟 Debug Mode。
- [x] Debug Mode 可查看重要進度旗標與線索解鎖狀態。
- [x] Debug Mode 提供 GDD 謎題 1–10 的統一章節快照與對應 App 跳轉。
- [ ] 後續逐步加入快速跳到 40／184／256，以及測試三個結局的功能。

**驗收：** 一般網址開啟時看不到左側面板；按下 `Ctrl + Shift + D` 或使用 `?debug=true` 時可顯示，再次按快捷鍵可隱藏，且不影響遊戲進度。

### B. 在手機內加入引導式線索 App `[ ]`

- [ ] 將玩家可用的線索功能放回手機世界內，而不是放在外部系統面板。
- [ ] App 顯示玩家已經發現的線索摘要。
- [ ] App 可顯示尚待追查的方向，但只提供引導，不直接給搜尋詞、密碼、高度序列或謎底。
- [ ] 未發現的線索應以疑問、模糊描述或待調查狀態呈現。
- [ ] 新提示必須由實際遊戲進度與玩家已取得的證據解鎖。

- [ ] 不重複其他 App 已完整呈現的內容，只保存可幫助玩家回想與繼續追查的摘要。

**驗收：** 遇到困難的玩家可以從 App 知道下一個調查方向，但不能只靠這個 App 直接得到任何主要謎題答案。

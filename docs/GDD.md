# SKG: Scorekeeper

> 副標題：**Nobody Was Supposed to Finish**  
> 本文件是本遊戲的北極星（North Star）。若實作、內容或美術方向與本文件衝突，優先保留「追查一款被覆蓋、被遺忘、卻仍然可能完成的遊戲」這個核心體驗。

## 1. 故事核心

十二年沒有更新的手機遊戲《Flappy Something》突然出現在商店更新列表中。更新後，它變成一款粗製濫造、充滿生成式素材痕跡的 Flappy Bird 仿作：介面廉價、圖片風格不一致、廣告按鈕四處閃爍，連原本的遊戲名稱都像被自動翻譯過。

所有正常玩家都會在第 37 分死亡。排行榜大部分分數也停在 37，只有少數帳號超過這個數字。第一名 ARC_184 的影片標題是：

> **I BROKE THE UNBEATABLE FLAPPY GAME — 184 POINTS WORLD RECORD**

ARC_184 聲稱自己使用 Internet Archive 保存的舊版本，任何找到舊裝置的人都能重現。主角起初只想證明 ARC_184 作弊、超越他並成為第一名，調查後卻發現排行榜真正隱藏的不是最高分，而是最低分：一筆只有舊版本才能讀取的異常紀錄 **−65535**。持有者是十二年前消失的開發者 Noah Kade；負分不是失敗，而是有人抵達遊戲真正終點後產生的 16 位元溢位。

## 2. 世界背景

### Lumen Arc

十二年前短暫存在的裝置，介於手機、掌上遊戲機與媒體播放器之間，擁有獨立應用程式格式、封閉商店與特殊高度感應器。因電池過熱風險被政府要求全面回收，裝置與專屬遊戲生態迅速消失。

### SKG

今日的 **SKG Automation** 是由機器生成內容的自動化服務商，經營廣告投放、自動內容生成、舊應用程式重新上架、品牌資產管理、無人客服與數位遺產貨幣化。十二年前它叫 **Silver Kite Games**，由技術負責人兼 CEO Elias Vale 與遊戲設計師 Noah Kade 創立。

《Skyline 256》是如今被重新命名、重新包裝的飛行遊戲。它原本有 256 個障礙區段與明確終點，並非無限得分遊戲。

## 3. 主要人物

### 主角

極度在意遊戲公平性的年輕玩家。初始目標是找出 ARC_184 的作弊證據並成為排行榜第一名。調查後才發現，母親 Mara 曾將舊裝置資料移植到目前手機，而 Noah Kade 正是自己失蹤多年的父親；主角不小心讓新版更新覆蓋了舊版。

### ARC_184

使用舊版本以低空路線通過第 37 分的第一名玩家。他故意用「作弊爭議」重新吸引玩家注意這款被遺忘的遊戲，並非真正反派。

### Noah Kade

《Skyline 256》的主要開發者。他不接受作品從未有人完整通關，在最後更新中加入秘密路線並親自完成遊戲；完成標記寫入 16 位元排行榜欄位後，錯誤顯示為 **−65535**。

### Mara

主角的母親，也是 Noah 的伴侶與 Lumen Arc 忠實使用者。她曾購買數百份遊戲序號支持 Noah，並替未來的孩子留下一台 Lumen Arc。多年後記憶衰退，只留下零碎對話、密碼提示與舊資料。

## 4. 核心體驗與設計支柱

1. **先讓玩家相信這是廉價遊戲，再讓玩家發現它值得被保存。**
2. **玩家透過手機中的應用程式、截圖、網頁快照與私人訊息拼出真相。**
3. **高分不是終點；完成、記憶與保存才是終點。**

遊戲不是突然變高級，而是剝開覆蓋其上的十二年：生成式廉價素材逐層消失，露出更原始、更真實的像素美術、測試文字、開發註解與被刪除的玩家資料。

## 5. 核心遊戲循環

1. 在手機模擬器中查看異常線索。
2. 在 ViewTube、Browser、AmazeMart、FaceSpace、Messages 等應用程式之間搜尋、比對與解讀資料。
3. 將線索轉成密碼、裝置、舊版本或飛行高度路線。
4. 回到飛行遊戲驗證推論，突破下一個不可能的門檻。
5. 重新理解分數、排行榜與「完成」的意義。

## 6. 主要控制與互動

- 手機桌面：點擊應用程式、返回、切換頁面與帳號。
- 謎題介面：搜尋、閱讀、放大、旋轉、排列截圖、輸入密碼。
- 飛行遊戲：點擊／按鍵控制上升，不操作時自然下落。
- 隱藏路線：依序輸入或維持高度 `184, 172, 149, 133, 121, 118, 126, 143`。
- 最終障礙：在第 256 區段使用高度 `0`，即不再點擊，讓小鳥落下穿過黑牆下方。

## 7. 完整流程與謎題

### 關卡分層：Level 1 與看得見卻進不去的 Level 2

- **0–36 分是 Level 1。** 這一段保留容易理解的 Flappy Bird 基礎操作，難度友善、規則單純，讓大多數玩家能穩定接近第 37 關。
- **玩家抵達第 36 分附近時，必須已經看得見第 37 關後仍有新的世界。** 可以透過 `LEVEL 2` 標示、遠方不同的場景、陌生障礙物剪影或難度升級提示，讓玩家確信遊戲並沒有在 37 結束。
- 第 37 關是 Level 1 與 Level 2 的封鎖邊界。正常飛行無法跨越；玩家應產生「明明後面還有內容，為什麼我就是進不去？」的疑問，而不是把 37 當成普通的遊戲終點。
- Level 2 只能在玩家完成手機調查、取得高度序列並通過秘密路線後進入。突破 37 是揭開被掩埋遊戲本體的瞬間，不是一般難度曲線中的普通升級。
- **延後實作的 Level 2 障礙構想：** 帶刺的水管水平面、可預測地上下移動的水管，以及會先給徵兆再從寬闊洞口咬出的巨大嘴花。這些機制用來恢復更強的 Flappy Bird 操作感，但不得提前出現在 0–36，也不得造成玩家在理解 37 的謎題前隨機死亡。
- Level 2 的危險必須可觀察、可預測；每次死亡都應讓玩家理解一條規則。普通水平面仍可安全停留，只有外觀清楚標示的刺、怪物或其他特殊表面才致命。

### 序章：廉價遊戲

玩家開啟更新後的《Flappy Something》。前三次在第 37 分死亡；第三次後排行榜自動展開，顯示大量 37、少量 38–62、ARC_184 的 184，以及 `65,535 ENTRIES HIDDEN`。鏡頭縮小，主角說：「這不是紀錄。這是作弊。」真正標題出現。

### 謎題 1：尋找第一名

在 ViewTube 搜尋 `ARC_184`。影片顯示第 37 分的奇怪低空路線；留言指出 ARC_184 使用 Archive 保存的舊版。他的回答是：`The one they took away from everyone.`

### 謎題 2：尋找舊版本

在 Archive 找到 `Skyline256_LAOS_Final.ipa`：十二年前上傳、平台 LAOS、相容 Lumen Arc、現代裝置無法執行，並註記 `Requires native altitude sensor`。玩家想起母親的舊裝置，但新版已覆蓋舊版。

### 謎題 3：購買舊裝置

在 AmazeMart 購買 Lumen Arc，卻收到一疊螢幕截圖。玩家可放大、旋轉與排列圖片，從舊桌面圖示、商店帳號 `SilverKite_Games` 得知遊戲原名是 **SKG: Skyline 256**。

### 謎題 4：解讀 SKG

搜尋結果包含 Smart Kitchen Group、Secure Key Gateway、Skyline Knowledge Grid、Sustainable Kinetic Goods 與 SKG Automation。玩家依產業、年份與圖示排除結果，在 Archive 將 SKG Automation 網站回溯到舊版 Silver Kite Games 頁面，發現 Lumen Arc 合作關係與 Noah Kade。

### 謎題 5：被機器覆蓋的公司

逐年查看網站快照：手寫開發日誌逐漸變成投資失敗公告、罐頭客服與自動化模板。舊文提到 Noah 正在製作「具有明確終點的飛行分數遊戲」，與當前無限分數玩法矛盾。

### 謎題 6：開發者的社群帳號

在 FaceSpace 將 Noah Kade 的貼文排序切換成最早發布，讀到 Lumen Arc 衰退、回收、Noah 的最後更新，以及 Mara 的留言：「別擔心，親愛的。我買了幾百份來支持你。我甚至替我們未來的孩子留了一台。」

### 謎題 7：最喜歡的數字

Noah 的舊問答回答最喜歡的數字是 `184-37-256`。玩家在 Messages 讀到母親的提示：「你爸爸總說數字不是密碼。它們是路。」

### 謎題 8：登入母親的舊帳號

提示為 `His favorite number, but not as a score.`。玩家從開發介面的 `Altitude / Gate / End` 將數字解讀為高度 184、障礙區 37、終點 256，得到密碼：`ALT184GATE37END256`。

### 謎題 9：母親與 Noah 的對話

私人訊息揭露：排行榜與伺服器即將關閉；Noah 在最後更新加入秘密通關路線；Lumen Arc 高度感應器在特定節奏下會產生誤差，使玩家穿過原本會碰撞的牆；他希望有人自己發現。最後留下：「如果他有一天找到這裡，別告訴他答案。讓他自己走完。我已經替他留下高度。」

### 謎題 10：找到名字中的路線

開發者帳號 `NK_184.172.149.133.121.118.126.143` 是第 37 分附近的飛行高度序列。玩家開啟隱藏開發者資訊介面，依序通過八個高度；第六個高度時碰撞判定短暫失效，小鳥穿過牆體並首次超過 37 分。

## 8. 隱藏關卡與排行榜真相

超過 37 後，天空變暗、雲停止、廣告消失，生成式圖片逐層剝落，露出原始像素美術。沿途出現未完成障礙物、測試文字、刪除的玩家名稱、舊公告、回收通知、伺服器關閉倒數與 Noah 的版本註解。

超過 ARC_184 的 184 分後，排行榜提示：

```text
SORT ORDER INVALID
SIGNED VALUE DETECTED
SHOW HIDDEN ENTRIES?
```

玩家往下瀏覽數千、數萬筆 0 分紀錄，最底部看見：`NOAH_KADE — −65535`。Noah 才是第一個完成遊戲的人；完成狀態 `65537` 寫入 16 位元有號欄位後發生溢位，因此被排在所有人之下。

## 9. 真正的 256 關

185 分後不再顯示排名，只顯示 `DISTANCE TO END`。第 256 個障礙是寫著 `SERVICE TERMINATED` 的垂直黑牆。玩家必須使用最後高度 `0`，完全不點擊，讓小鳥落下穿過螢幕底部，進入隱藏完成區域。

通關畫面顯示：

```text
SKYLINE COMPLETE
THANK YOU FOR PLAYING
```

分數欄位跳動：`256 → 65535 → −65535`。

## 10. 結尾與最後選擇

Credit 依序列出 Elias Vale、Mara、ARC_184 真實姓名、購買序號的帳號與主角幼年暱稱，最後是 Noah 的版本註解：這不是一款無限遊戲；裝置、商店與伺服器都會消失，他能做的只是替它留下最後一關。

通關後玩家選擇：

1. **提交分數**：把完成紀錄轉為 257 分成為第一名，但 Noah 的負分仍被忽略。`You got the high score.`
2. **公開秘密**：公開路線與故事，玩家重新下載，但 SKG 移除舊版資料並關閉排行榜。`Everyone knew. Nothing remained.`
3. **保存遊戲**：不提交分數、不公開完整答案，保存可執行舊版、技術文件、排行榜資料與證據；第一位下載者是 ARC_184。`A game does not need to run forever. It only needs to remain possible.`

第三項是最接近真結局的選擇。

## 11. 主題

- **High Score Chaser**：主角追第一、ARC_184 追關注、Noah 追完成、玩家追真相；高分不是答案。
- **Stop Killing Games**：更新覆蓋舊版、硬體消失、商店與伺服器關閉、公司轉型抹去創作者、資料格式讓紀錄隱藏；「擁有遊戲」不代表未來仍能執行它。
- 核心提問：誰有資格只因遊戲廉價、失敗或過時，就決定它不值得留下？

## 12. 玩家體驗節奏

- 0–3 分鐘：廉價、令人失望的 Flappy Bird 表層。
- 3–8 分鐘：發現 37 分必死、排行榜異常與世界紀錄。
- 8–18 分鐘：尋找舊版本、購買假裝置、從截圖找到舊名稱。
- 18–30 分鐘：追查 SKG、Archive 與開發者帳號。
- 30–42 分鐘：發現 Mara 與 Noah 的關係並登入舊帳號。
- 42–50 分鐘：破解高度數列並穿過第 37 分。
- 50–60 分鐘：探索遊戲殘骸、超越 184、找到 −65535。
- 60–65 分鐘：抵達 256、觀看 Credit、做出保存選擇。

Jam 版可壓縮至約 25–35 分鐘，但必須保留：37 分必死、手機調查、舊版本／Noah 線索、隱藏飛行路線、−65535 真相，以及三選一結局。

## 13. MVP 與裁切原則

### Must-Have

- [ ] 手機桌面與至少五個可互動應用程式
- [ ] 37 分必死的飛行遊戲表層
- [ ] ViewTube、Archive、AmazeMart、FaceSpace、Messages 的最小謎題鏈
- [ ] `ALT184GATE37END256` 密碼與高度序列
- [ ] 突破 37、排行榜 `−65535` 與 256 完成段落
- [ ] 三個結局選項，保存遊戲為真結局方向

### Nice-to-Have

- [ ] 可旋轉排列的完整截圖互動
- [ ] 多期網站快照與廣告素材剝落效果
- [ ] 原始像素美術逐層恢復
- [ ] 完整 Credit 與更多刪除玩家資料
- [ ] 高度感應器／螢幕刻度的沉浸式呈現

### 第一批裁切

若時間不足，先合併或以靜態畫面取代部分應用程式；不可裁掉 37 分、ARC_184、Noah、`−65535`、256 終點與保存選擇，因為它們共同構成北極星。

## 14. 實作風險與 fallback

- 應用程式過多導致內容爆量：先使用固定線索頁與最小點擊路徑。
- 飛行高度序列難以做出可靠輸入：先用可驗證的高度檢查／節奏窗口模擬感應器。
- 排行榜與溢位效果難以呈現：以明確的排序錯誤畫面與固定 `−65535` 紀錄保留敘事。
- 25–35 分鐘內資訊過密：保留一條主線，將額外背景放到可選的快照、留言與 Credit。

## 15. 目前開發北極星

先做出一個玩家能走完的最小版本：**打開廉價遊戲 → 在第 37 分死亡 → 追查 ARC_184 與舊版本 → 取得高度路線 → 穿過 37 → 看見 Noah 的 −65535 → 抵達 256 → 選擇是否保存。**

任何新功能都必須回答：它是否讓玩家更清楚地感受到「一款遊戲如何被覆蓋、消失，卻仍然可能被保存」？若沒有，延後到 MVP 之後。
